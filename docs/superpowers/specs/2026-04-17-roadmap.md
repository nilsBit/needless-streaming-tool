# Stream-Toolkit Roadmap: Nische vertiefen

**Date:** 2026-04-17
**Status:** Approved

## Strategie

Stream-Toolkit positioniert sich als **Projekt-Management + Stream-Management** Tool. Es ergänzt StreamElements/Streamlabs, ersetzt sie nicht. Statt in die Breite zu gehen (Alerts, Loyalty, Song Requests), vertiefen wir die Kernstärke und polieren die UX.

**Zielgruppen:** Coding-Streamer, Art-Streamer, Coworking/Study-Streamer, Game Dev

## Phasen

### Phase 1: i18n fertig machen
**Status:** Plan geschrieben, bereit zur Umsetzung
**Spec:** `docs/superpowers/specs/2026-04-17-i18n-completion-design.md`
**Plan:** `docs/superpowers/plans/2026-04-17-i18n-completion.md`

- ~120 hardcoded Deutsche Strings → `t()` System
- Englische Übersetzungen für alle Panels + Help-Docs
- Sprachwahl im Onboarding-Wizard (Step 0)
- Help-Docs pro Sprache (separate Dateien)
- Vorbereitung für weitere Sprachen (nur neue Datei + Keys nötig)

**Aufwand:** Klein-Mittel | **Impact:** Hoch (öffnet internationale Nutzerbasis)

---

### Phase 2: UX Polish / Onboarding
**Status:** Nicht gestartet

- Onboarding-Wizard UX verbessern (Fortschrittsanzeige, Validierung)
- First-Use Experience optimieren (leere States, Hinweise)
- Tastatur-Navigation verbessern
- Responsive Layout für verschiedene Fenstergrößen
- Error States und Loading States konsistent machen
- Tooltips für Overlay-URLs und API-Endpoints

**Aufwand:** Klein | **Impact:** Mittel (senkt Einstiegshürde)

---

### Phase 3: Overlay-Anpassbarkeit
**Status:** Nicht gestartet

- UI im Settings-Tab zum Anpassen von Overlay-Parametern (Farben, Fonts, Positionen, Größen)
- Overlay-Settings pro Overlay als JSON in der DB speichern
- Overlays laden ihre Config via Public API Endpoint
- Live-Preview im Editor
- Kein HTML-Upload nötig für einfache Anpassungen

**Aufwand:** Mittel | **Impact:** Hoch (macht alle 8 Overlays sofort wertvoller)

---

### Phase 4: Zeiterfassung pro Feature
**Status:** Nicht gestartet

- Timer an einzelne `project_items` koppeln (nicht nur globaler Challenge-Timer)
- Automatische Zeiterfassung wenn ein Item auf "in_progress" gesetzt wird
- Overlay zeigt aktuelles Feature + verstrichene Zeit
- Stats-Dashboard: Zeitübersicht pro Feature/Session
- Export: CSV mit Feature + Zeitaufwand

**Aufwand:** Klein | **Impact:** Mittel (baut auf bestehendem Timer-System auf)

---

### Phase 5: Kanban-Board
**Status:** Nicht gestartet

- Upgrade des Progress-Trackers zu einem Drag-and-Drop Kanban
- Spalten: Backlog → In Progress → Done (konfigurierbar)
- Sortierung per Drag
- Overlay zeigt aktuelle Spalte + Items
- Optional: WIP-Limits pro Spalte

**Aufwand:** Mittel | **Impact:** Hoch (Kernfeature der Nische, visuelles Upgrade)

---

### Phase 6: GitHub/Linear Issues Import
**Status:** Nicht gestartet

- GitHub Issues aus einem Repo importieren → werden zu `project_items`
- Linear Issues aus einem Projekt importieren
- Einmaliger Import oder periodischer Sync
- Mapping: GitHub Label → Kanban-Spalte
- Auth via Personal Access Token (Settings)

**Aufwand:** Klein-Mittel | **Impact:** Hoch (Killer-Feature für Dev-Streamer)

---

### Phase 7: Overlay Themes
**Status:** Nicht gestartet

- Vorgefertigte visuelle Presets für Overlays
- Themes: "Coding" (Terminal-Style), "Art" (Watercolor/Brush), "Coworking" (Clean/Minimal), "Gaming" (NFS-Unbound-Style = aktuell)
- Theme wählbar pro Overlay oder global
- Theme = CSS-Variablen-Set, nicht komplett neues HTML
- Community kann Themes teilen (JSON-Export/Import)

**Aufwand:** Mittel | **Impact:** Mittel (spricht neue Zielgruppen visuell an)

---

### Phase 8: Auto-Highlight Clips
**Status:** Nicht gestartet

- Automatische Erkennung von Clip-würdigen Momenten basierend auf:
  - Chat-Aktivität (Spikes in Messages/Emotes)
  - Channel Point Redemptions
  - Raid-Events
  - Milestone-Completions
- Auto-generierte Clip-Marker mit Confidence-Score
- Streamer reviewed und bestätigt/verwirft
- Integration mit bestehendem Clip-Export (DaVinci CSV)

**Aufwand:** Groß | **Impact:** Hoch (einzigartig, kein Konkurrent bietet das)

---
